# Ultimate-Council

Invoke the `council` skill in **ultimate-council** mode: run the normal 5-voice council AND force all THREE expert witnesses.

- Force the UltraOracle by setting `ULTRA_ORACLE_COUNCIL_FORCE=1` (a plain, non-exported assignment) at the top of the council's single Step 4 dispatch Bash block, `unset` at the end (see Step 4.5).
- Force the Mythos Witness by setting `_forced=1` in the Step 4.6 gate pre-check block ‚Äî a **single** block now (ADR 0019 deleted the gateway-fallback block that used to need its own copy). The gate then dispatches an `Agent(model="fable")` Mythos Witness ‚Äî in-harness, in-account, no gateway creds ‚Äî as the only transport (see Step 4.6).
- Force the Mechanism Witness by **flipping the literal `MECHANISM_WITNESS=0`** already in the Step 4b preamble to `MECHANISM_WITNESS=1` ‚Äî **but ONLY when the Step 4.6 gate returned `MYTHOS_ATTEMPT=1`.** (Edit the literal in place; do NOT add a separate assignment, because that literal exists precisely to shadow any repo-injected ambient `MECHANISM_WITNESS`.) The Mechanism Witness shares the Mythos Witness's authorization (that same gate), so `BUSDRIVER_ULTIMATE=0` ‚Äî which makes the gate return `MYTHOS_ATTEMPT=0` ‚Äî MUST suppress k3 too. If `MYTHOS_ATTEMPT=0`, leave the literal at `0` (the block skips k3), exactly as you skip the fable subagent. The self-gated Step 4b opencode dispatch block fires kimi-k3 only when the literal is `1` (see Step 4.7). Never `export` it. See ADR 0027.

For the UltraOracle's `ULTRA_ORACLE_COUNCIL_FORCE=1`: never subshell, never `export`, never a `VAR=1 cmd` prefix (a trigger phrase alone sets no env var, and a one-command prefix would not persist to the later gate checks in the same block). The UltraOracle dispatches via the separate `ultra_oracle_consult` adapter (the `oracle` CLI's ChatGPT Pro browser engine, see `skills/council/SKILL.md` Step 4.5). The Mythos Witness is a **fable subagent only** ‚Äî no gateway, no creds, no metered billing, and no `ULTIMATE_COUNCIL_FORCE` plumbing (that existed solely to authorize the deleted gateway helper). A failed subagent renders `MYTHOS_FAILED` directly; there is no second transport (see `skills/council/SKILL.md` Step 4.6).

Render each witness in its OWN separate section ‚Äî `## UltraOracle ‚Äî Expert Witness [ORACLE_SUMMARY_REVIEW]`, then `## Mythos Witness ‚Äî Expert Witness`, then `## Mechanism Witness ‚Äî Expert Witness [claim-vs-mechanism]` ‚Äî AFTER the five voices and BEFORE the Verdict. All three are EXCLUDED from the vote tally; their claims are unverified-until-checked (grep/Read/run). On timeout/empty/creds-missing render a loud `ORACLE_FAILED` / `MYTHOS_FAILED` / `MECHANISM_FAILED` banner (or `MECHANISM_ABSENT [opencode unavailable]` when opencode is missing), never a silent omission.

Enablement is user-config only (`~/.claude/busdriver.json` ‚Üí `ultimate.surfaces.council` for the Mythos Witness, `ultraOracle.council.enabled` for the UltraOracle); the Mechanism Witness shares the Mythos Witness's `ultimate.surfaces.council` authorization (same `MYTHOS_ATTEMPT` gate). The `*_FORCE=1` / `MECHANISM_WITNESS=1` vars are set only for an interactive ultimate-council request, never from repo/project config. See `skills/council/SKILL.md` Steps 4.5, 4.6, and 4.7.

`BUSDRIVER_ULTIMATE=0` is a global force-off that outranks the per-run `_forced=1` escape hatch ‚Äî the Step 4.6 gate pre-check tests `BUSDRIVER_ULTIMATE != 0` first, so if the operator has it set in their environment, forcing the Mythos Witness via an ultimate-council request will still be skipped. **Because the Mechanism Witness is gated on the SAME `MYTHOS_ATTEMPT` output, `BUSDRIVER_ULTIMATE=0` suppresses k3 too** (you only set `MECHANISM_WITNESS=1` when `MYTHOS_ATTEMPT=1`). It has no effect on `ULTRA_ORACLE_COUNCIL_FORCE`/the UltraOracle, which has no equivalent global force-off.
